Техническое задание: Кроссплатформенное приложение для скачивания аудио из плейлистов YouTube

   1. Цель проекта
   Создать десктопное приложение с графическим интерфейсом (GUI), которое принимает ссылку на плейлист YouTube, извлекает список треков, фильтрует их по заданным правилам, позволяет пользователю вручную скорректировать список и скачивает выбранные аудиодорожки в формате MP3 в указанную локальную папку. Приоритетная целевая ОС — Windows, но архитектура должна позволять сборку под macOS и Linux.
   2. Требования к интерфейсу (GUI)


* Главное окно приложения содержит текстовое поле для вставки ссылки на плейлист.
* Поле для выбора папки сохранения с кнопкой "Обзор" (открывает стандартное диалоговое окно ОС).
* Кнопка "Загрузить список" для анализа плейлиста.
* Центральная область: интерактивный список всех видео из плейлиста с чекбоксами (галочками) напротив каждого пункта.
* Кнопки быстрого выбора: "Выбрать все" и "Снять выделение со всех".
* Кнопка "Скачать" для запуска процесса.
* Визуальный индикатор прогресса (Progress Bar) и текстовая строка состояния.
* Окно или вкладка "История", где отображается список ранее скачанных треков, дата загрузки и пути к файлам.


   1. Функциональные требования

3.1. Анализ плейлиста и автоматическая фильтрация
После нажатия кнопки "Загрузить список" программа парсит плейлист и выводит список треков. При этом автоматически применяются следующие фильтры:

* Длинные видео: Программа должна визуально помечать или автоматически снимать галочки с очень длинных видео (более 10-15 минут), оставляя пользователю возможность вернуть галочку вручную.
* Определение "Slowed" версий: Если в названии видео или тегах присутствуют маркеры замедленных треков (например, "slowed", "slowed & reverb", "slowed down"), программа должна снять с него галочку, выдать предупреждение рядом с треком и предложить пользователю кнопку "Найти оригинал" (выполняет быстрый поиск на YouTube по названию без слова slowed для замены ссылки).

3.2. Формирование имен файлов и метаданных

* Имя итогового MP3-файла должно формироваться по шаблону: Исполнитель - Название трека.mp3.
* Определение исполнителя: Программа должна парсить имя автора (исполнителя) из названия канала, на котором выложено видео, либо использовать алгоритмы поиска оригинального исполнителя (например, через официальные аудио-теги YouTube Topic, если они доступны в метаданных видео через yt-dlp).
* Спецсимволы, запрещенные операционной системой в названиях файлов, должны автоматически удаляться или заменяться на пробелы с учетом специфика конкретной ОС (Windows/macOS/Linux).

3.3. Скачивание и конвертация

* Скачивание запускается только для отмеченных пользователем позиций.
* Все файлы на выходе должны быть строго в формате MP3 (качество не ниже 192-320 kbps).


   1. Логирование и история скачиваний


* Программа должна локально сохранять историю всех успешно скачанных файлов (название, автор, ссылка на YouTube, дата скачивания и папка сохранения).
* История должна храниться в легковесном формате (например, SQLite или JSON-файл в папке пользователя), чтобы данные не терялись после перезапуска приложения.


   1. Технические требования и кроссплатформенность


* Язык программирования: Python.
* Библиотеки для GUI: PyQt6 или PySide6 (обеспечивают нативную кроссплатформенность Windows/macOS/Linux).
* Инструменты для скачивания и анализа: актуальная библиотека yt-dlp.
* Конвертация: утилита FFmpeg. Для Windows-версии FFmpeg должен быть вшит в сборку приложения (в папку с программой), чтобы пользователю не приходилось устанавливать его отдельно в систему.
* Сборка проекта: Использовать PyInstaller или fbs для компиляции проекта в готовый исполняемый файл. Первоочередная задача — собрать рабочий .exe для Windows. При необходимости код должен без изменений компилироваться под macOS и Linux.


   1. Обработка ошибок


* Программа должна игнорировать заблокированные, приватные или удаленные видео, не прерывая общую очередь скачивания.
* Вывод понятного уведомления об ошибке, если вставлена некорректная ссылка.
* Вывод сообщения "Скачивание успешно завершено" по окончании работы.
